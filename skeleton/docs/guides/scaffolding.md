# Scaffolding Guide: Clone → Install → Add Modules

This guide walks through seam 8 — the complete scaffold path from clone to a running project with optional modules.

## Quick Start: Clone → Run

```bash
# Clone this skeleton into your project
git clone <repo-url> my-project
cd my-project

# Install dependencies
bun install

# Run the full CI pipeline (typecheck + boundary checks + tests)
bun run ci

# Start development
bun run dev
```

This gets you a working CRUD example (tenancy isolation, auth, error handling) in seconds. See `apps/api` and `apps/web` for the running example.

## Architecture

The skeleton ships with:

- **`packages/contract`** — Zod schemas for all operations (forms, API routes, OpenAPI docs, client generation)
- **`packages/core`** — Pure business logic (no I/O)
- **`packages/db`** — Drizzle + PGlite for local dev, Postgres-compatible for production
- **`apps/api`** — Hono HTTP server with Zod-validated routes; one worked example (Invoices)
- **`apps/web`** — Vite + React + TanStack Router/Query; one worked feature slice (Invoices list/create/edit/delete)

**No hand-written API clients.** The web app imports generated types from `bun run gen:client` (contract → OpenAPI → types).

## Adding Modules

Modules are optional, copyable packages that extend the skeleton. The repo ships with:

- **`modules/money`** — Money value objects (decimal.js internals, allocation, multi-currency)
- **`modules/queue`** — Async job queues (pg-boss on Postgres/PGlite, CF Queues on Workers)

### From the Original Repository

If you cloned the skeleton directly and later want to add modules from the kongmy-stack repo:

```bash
# From the original kongmy-stack repo root, pointing to your cloned project
bun scripts/add.ts money --into /path/to/my-project

# Then in your project:
cd /path/to/my-project
bun install
bun run test
```

### Pre-Seeded in Your Clone

If you cloned an archetype that includes modules, they're already in `packages/<module>` and integrated into workspaces.

## Testing

```bash
# Run all tests (db + api + web)
bun run test

# Test a specific package
bun run --cwd packages/money test
bun run --cwd apps/api test

# Run the acceptance suite (Playwright smoke + module verifies)
bun run acceptance
```

## Contract-First Development

Every operation starts with a contract schema in `packages/contract/`:

```typescript
// Define: packages/contract/resources/invoice.ts
export const Invoice = z.object({
  id: id('inv'),
  amount: z.number().positive(),
  // ...
})

export const listInvoices = listResponse(Invoice)
export const createInvoice = action('create', Invoice)
```

From this one definition:
1. **API route** gets auto-validated input/output
2. **OpenAPI spec** generates automatically
3. **Frontend types** are generated by `bun run gen:client`
4. **Zod form validation** works everywhere

No route without a contract. No hand-written client types.

## The 8 Seams

**Seam 8** is the scaffold path you're on right now. The full 8 seams:

1. **Contract → Generated Client** — `packages/contract` defines operations, `bun gen:client` produces typed client
2. **Query factories** — Per-resource `queryOptions` for TanStack Query
3. **DataTable pagination** — One set of `limit`/`offset`/`sort` names across API, UI, and URL
4. **Zod form validation** — `zodResolver(contract.CreateInvoice)` wires contract to React forms
5. **Route search params** — Contract schemas validate URL parameters (shareable URLs are valid API queries)
6. **Error mapping** — AppError envelope → form field errors or toast
7. **Auth** — Server session context + client session hook + route guards
8. **Scaffold** — Clone → install → optional modules → zero platform code written (you're here)

## Gotchas

### Node Modules Cache Invalidation

After adding a module:

```bash
# Vite caches dependencies; clear it:
rm -rf node_modules/.vite apps/web/node_modules/.vite
bun install
```

### TypeScript Path Aliases

The root `tsconfig.json` defines paths for all packages:

```json
{
  "compilerOptions": {
    "paths": {
      "@kongmy-stack/contract": ["packages/contract/dist"],
      "@kongmy-stack/core": ["packages/core/dist"],
      "@kongmy-stack/db": ["packages/db/dist"],
      "@kongmy-stack/money": ["packages/money/dist"]
    }
  }
}
```

If you add a module, update `tsconfig.json` and ensure the package is in workspaces.

### Dependency Versions

Modules list their exact dependencies (e.g., `zod ^4.0.0`). On `bun install`, Bun resolves to the highest compatible version in the workspace. If a module needs a specific version, it's pinned in its `package.json`.

### Environment Variables

Copy `.env.example` to `.env.local` and fill in secrets:

```bash
cp .env.example .env.local
```

The API reads from `.env.local` at startup. Web app environment variables are bundled at build time.

## Troubleshooting

### `bun run ci` Fails

Usually a typecheck or boundary issue. Run individually:

```bash
bun run type-check      # TypeScript across all packages
bun run boundary-check  # Dependency cruiser (allowed imports)
bun run test            # Test suite
```

### `bun install` Doesn't Update a Module

If you modified a module's source:

```bash
rm -rf node_modules packages/*/node_modules pnpm-lock.yaml
bun install
```

### Web App Won't Start

```bash
rm -rf apps/web/node_modules/.vite
bun run dev
```
